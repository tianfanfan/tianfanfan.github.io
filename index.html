文档工具 ​

简介 ​

本工具是一个文档生成工具，可以将 Markdown 编写的内容，生成交互丰富的文档，并生成可以轻松部署到任何地方的 HTML 网页。

文档兼容性 ​

工具底层使用的是 VitePress，语法兼容所有 Markdown 语法。

支持所有 VitePress 的配置和扩展语法。可参考 VitePress。

另外新扩展了 Markdown 的插件，支持 新的一些语法。

安装与启动 ​

前置准备 ​

  • Node.js 16 及以上版本
  • 支持 Markdown 语法的编辑器

安装 ​

使用以下方式安装它

shell
# 安装，其他的安装 npm 全局包方案也可以
npm i -g @tencent/toxdoc

# 不指定文件夹，后续将会询问
toxdoc init

# 指定文件夹，默认是 document
toxdoc init -d document

运行 ​

sh
# 在 document 文件路径下，命令行启动
npm run dev

文件结构 ​

假设选择在 document 中搭建文档，生成的文件结构应该是这样的

document 目录作为站点的项目根目录。

docs 内部接口可以参考 VitePress 路由章节

sh
.
└─ document
   ├─ docs
   │  ├─ .vitepress
   │  // 文档 markdown 存放地址
   │  ├─ guide
   │  │  ├─ getting-started.md
   │  │  └─ index.md
   │  ├─ index.md
   │  └─ prologue.md
   ├─ package.json
   └─ .prettierrc.cjs

生成的 HTML 页面会是这样：

sh
index.md                  -->  /index.html (可以通过 / 访问)
prologue.md               -->  /prologue.html
guide/index.md            -->  /guide/index.html (可以通过 /guide/ 访问)
guide/getting-started.md  -->  /guide/getting-started.html

生成的 HTML 可以托管在任何支持静态文件的 Web 服务器上。

tox doc 扩展语法 ​

步骤条导航 ​

如您当前看到的文档，即是用此工具构建出，文档会自动生成右侧步骤导航，仅需要注意 Markdown结构。

此功能做了以下几件事

  1. 自动识别 H1 为大标题，这需要你保证一个文件中只有一个 H1。
  2. 将 H2 到 H2 之间的内容识别为 Step 步骤模式识别 H2 内容为 Step 标题。
  3. 没一个 Step 底部会出现 Back 和 Next。

如果要关闭此行为，请在顶部配置

yaml
---
step: false
---

小屏幕优化 ​

在手机屏幕上，右侧步骤条将自动隐藏。如果需要适配手机，则需要配置。 请在 config 书写以下代码，含义是在顶部导航标题显示 H1 , H2 标签。

yaml
---
outline: [1, 2]
---

shell

npm run build

# 预览
npm run docs:preview

构建产物在 document/docs/.vitepress/dist。可以用任何静态部署方案部署该目录。
