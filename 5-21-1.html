函数如何划分类型 ​

代码大全 7.6 章节

https://juejin.cn/post/7189974156576817207

现代的编程语言，如 C++、Java、Visual Basic 等，都同时支持函数和过程。函数是指有返回值的子程序;

过程是指没有返回值的子程序。在 C++ 中，通常把所有子程序都称为“函数”:

然而，那些返回值类型为 void 的函数在语义上其实就是过程。

函数与过程的区别更多的是语义的区别，而不是语法的区别，你还是要以语义为准。

如果函数执行，返回了内容，它叫函数。

如果函数执行，没有返回内容，它叫过程。

衍生下 ​

纯函数

js
(a, b) => a + b;

副作用 effect

js
(a, b) => {
  // 并不是说，这个函数是副作用，而是它调用的 console.log 是副作用
  console.log(a, b);
};

结合一下 react ​

js
const App = () => {
  return <>1</>;
};

思考下这段代码有多少个函数相关？

正确答案 ​

js
const App = () => {
  return <>1</>;
};

第一段 ​

看起来只有一个函数，其实有很多函数。

首先， const App 就是一个函数，它是一个 过程 + 有返回值函数， 申请了一个指针，且把指针的操作权给了你。

第二段 ​

js
() => {
  return <>1</>;
};

简单点看， 这一段是一个较纯的函数，它没有输入，输出也是固定的。

第二段真的是纯函数么？ ​

思考下，它的输出真的是固定的么？

所以它也不是纯函数。

第三段 ​

js
<>1</>

会被转化成 React.createElement

第三段 ​

最后一个赋值操作，也是一个过程，只不过，它变成了语法。不再是函数形式。它也是一个动作，副作用。

如果世界上只有纯函数，会怎样？ ​

世界将会是一幅画，无法有任何变化。

禁止的画面。时间禁止。

react 的思想 ​

js
const App = () => {
  useEffect(() => {
    document.title = 'title';
  });
  return <>1</>;
};

react 把过程，(副作用) 放到了 useEffect, 把其他位置全部做成了类似纯函数（实际不是）。但已经很接近了。
